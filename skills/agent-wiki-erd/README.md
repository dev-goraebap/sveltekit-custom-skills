# agent-wiki-erd

PRD나 도메인 설명을 입력하면 ERD(Entity-Relationship Diagram)를 Mermaid `.mmd` 파일로 생성하는 스킬이다. **agent-wiki**로 구성된 위키 환경이 전제 조건이며, 결과물은 `.wiki/design/`에 저장된다.

## 왜 만들었나

데이터 모델링은 설계 초기에 반드시 거치는 과정인데, 매번 도구를 따로 열거나 직접 다이어그램을 그리는 건 번거롭다. PRD나 도메인 설명을 이미 위키에 정리해두었다면, 그걸 그대로 입력으로 받아 ERD를 뽑아내는 게 자연스럽다고 생각했다.

Mermaid를 선택한 이유는 단순하다. 텍스트 기반이라 Git으로 버전 관리가 되고, VS Code에서 바로 프리뷰할 수 있다. 이미지 파일을 별도로 관리할 필요가 없고, 에이전트가 수정하기에도 텍스트가 훨씬 편하다.

개념·논리·물리 3종을 모두 지원하는 건, 설계 단계에 따라 필요한 추상화 수준이 다르기 때문이다. 처음엔 엔티티 관계만 잡고, 나중에 속성과 타입을 채우고, 최종적으로 DB 컬럼까지 확정하는 흐름을 하나의 스킬 안에서 자연스럽게 이어갈 수 있다.

## 고려할 점

- `.wiki` 심볼릭 링크가 없으면 동작하지 않는다. `agent-wiki` 스킬로 위키 환경을 먼저 구성해야 한다.

- 다크 테마 + handDrawn 스타일이 기본이다. 밝은 테마가 필요하면 생성된 `.mmd` 파일의 config 섹션을 직접 수정하면 된다.

- 엔티티가 많아지면 하나의 다이어그램으로 표현하기 어려울 수 있다. 이 경우 도메인 단위로 파일을 나누는 것이 현실적이다.

## 워크플로우

### ERD 생성

`ERD 만들어줘`와 같은 프롬프트를 요청한다. 에이전트가 입력(PRD, 도메인 설명, 테이블 목록 등)을 분석하여 데이터 모델을 도출하고, 원하는 수준(개념/논리/물리)을 확인한 뒤 `.wiki/design/`에 `.mmd` 파일을 생성한다.

### 리뷰와 수정

VS Code의 Mermaid 프리뷰 확장으로 결과를 확인하고, 엔티티 추가·삭제, 관계 변경, 속성 수정 등을 자연어로 요청하면 반영된다.

### 수준 전환

기존 ERD를 기반으로 수준을 올리거나 내릴 수 있다. 개념 ERD에서 시작해 논리, 물리로 점진적으로 구체화하는 흐름이 일반적이다.
