# agent-wiki

위키(Wiki)는 누구나 읽고 편집할 수 있는 지식 저장소다. **agent-wiki**는 그 독자를 사람뿐 아니라 AI 에이전트로 넓힌다. 여러 프로젝트에 흩어진 에이전트들이 하나의 지식 저장소를 함께 바라볼 수 있도록, 심볼릭 링크 기반의 연결을 도와주는 스킬이다.

## 왜 만들었나

핵심은 바이브 코딩을 더 효율적으로 하기 위함이다.

하나의 제품을 만들더라도 상황에 따라 레포가 여러 개로 나뉘는 경우가 흔하다. Next.js나 SvelteKit 같은 풀스택 프레임워크를 쓰다가 API 서버를 분리한다거나, 별도의 모바일 앱을 만들어야 하는 경우가 그렇다. 이렇게 멀티레포 환경이 되면 설계 문서를 어디에서 관리할지가 애매해진다. 프로젝트마다 따로 두면 중복이 생기고, 그렇다고 위키 저장소를 별도로 만들자니 문서를 확인하거나 수정할 때마다 파일 탐색기나 에디터를 따로 열어야 하는 번거로움이 있었다.

처음에는 각 작업 환경에서 `../[project]-wiki`를 참조하는 지침을 두어 해결했다. 에이전트가 문서를 컨텍스트에 올리는 것 자체는 가능했지만, 내가 직접 문서를 검토하거나 변경 요청을 하려면 결국 별도의 창을 열어야 했다.

그래서 심볼릭 링크를 활용하기로 했다. 원본 위키 저장소를 각 프로젝트의 `.wiki` 폴더에 링크로 연결하고, Git에서는 추적하지 않도록 설정하면 에이전트도 사람도 프로젝트 안에서 바로 문서에 접근할 수 있다. 이 방식을 반복적으로 쓰다 보니 설정 과정 자체가 번거로워져서 스킬로 만들게 되었다.

## 고려할 점

- 이 스킬은 저장소를 만들고 심볼릭 링크로 연결하는 것까지만 담당한다. 어떤 문서를 넣고 관리할지는 작업자마다 중요하게 여기는 문서가 다르기 때문에 직접 결정하는 것이 맞다고 본다.

- 팀 협업까지 고려한 도구는 아니다. 에이전트 위키를 Git으로 올려서 공유할 수는 있지만, 협업 과정에서 생기는 충돌 해소나 편리한 자동화까지는 제공하지 않는다. 어디까지나 로컬 환경에서 여러 프로젝트의 에이전트들이 하나의 지식 저장소에 쉽게 접근할 수 있게 하는 데 목적이 있다.

- 위키의 모든 문서를 항상 컨텍스트에 올려두는 것은 비효율적일 수 있다. 예를 들어 ERD나 Sitemap 같은 초기 설계 문서는 어느 정도 MVP가 완성된 뒤에는 소스코드에 이미 반영되어 있기 때문에 굳이 매번 올릴 필요가 없다. 반면 아키텍처 원칙이나 구현 패턴처럼 일관성을 유지해야 하는 문서는 상시 로드해둘 가치가 있다. 프로젝트 단계에 맞춰 어떤 문서를 언제 올릴지, 컨텍스트 엔지니어링을 통해 조율하는 것이 바람직하다고 생각한다.

## 워크플로우

### 새로운 지식 저장소 생성

작업 중인 프로젝트 환경이나 비어 있는 환경에서 `/agent-wiki` 슬래시 명령어 또는 `위키 저장소 만들어줘`와 같은 프롬프트를 요청한다. 에이전트가 현재 맥락을 파악하고, 소스코드가 있는 환경이라면 상위 디렉토리와 같은 레벨에 `[프로젝트명]-wiki`라는 이름으로 별도의 저장소 생성을 제안한다.

프로젝트 환경일 경우 루트에 `.wiki` 폴더를 만들고, 생성된 지식 저장소를 심볼릭 링크로 연결한다. `.wiki` 폴더는 Git으로 추적할 필요가 없으므로 `.gitignore`에 자동으로 등록된다.

> [!TIP]
> 저장소를 만든 뒤 무엇부터 채워야 할지 고민된다면, github에서 만든 [prd 스킬](https://github.com/github/awesome-copilot)을 활용해보자. `npx skills add github/awesome-copilot --skill prd`로 설치하면 제품 요구사항 문서(PRD)를 대화형으로 생성할 수 있다. 위키의 첫 문서로 좋은 출발점이 된다.

### 기존 지식 저장소 연결

이미 지식 저장소가 존재하고 다른 작업 환경에 연결하려는 경우, `위키 저장소 연결해줘`와 같은 프롬프트를 요청한다. 에이전트가 상위 디렉토리에서 기존 위키 저장소를 탐색하여 연결한다. 별도의 위치에 있는 경우에는 직접 경로를 지정할 수도 있다. 위키 저장소의 폴더명을 변경했거나 심볼릭 링크가 깨진 경우에도 같은 방식으로 다시 연결할 수 있다.

### 위키 변경사항 커밋

프로젝트에서 `.wiki`를 통해 문서를 추가하거나 수정한 뒤, `위키 커밋해줘` 또는 `문서 커밋해줘`와 같은 프롬프트를 요청한다. 에이전트가 `.wiki` 심볼릭 링크를 따라 원본 위키 저장소로 이동하여 변경사항을 확인하고, 내용을 분석한 뒤 깔끔한 커밋 메시지를 작성하여 커밋한다. 소스코드 프로젝트와 위키 저장소는 별도의 Git 저장소이므로, 위키 문서의 변경 이력이 소스코드 커밋과 섞이지 않고 독립적으로 관리된다.
