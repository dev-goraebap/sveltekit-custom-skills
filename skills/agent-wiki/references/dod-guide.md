# Definition of Done (DoD) 가이드

Definition of Done은 User Story가 "완료"로 인정받기 위해 충족해야 하는 공통 기준이다. 프로젝트 전체에 적용되는 품질 게이트 역할을 한다.

---

## 출력 경로

```
wikicraft/definition-of-done.md
```

---

## 템플릿

```markdown
# Definition of Done

## 개요

이 문서는 모든 User Story가 "Done" 상태로 전환되기 위해 충족해야 하는 공통 완료 기준을 정의한다.

## 코드 품질

- [ ] 코드 리뷰 완료 (최소 1명 승인)
- [ ] 정적 분석 / 린트 경고 없음
- [ ] 코딩 컨벤션 준수

## 테스트

- [ ] 단위 테스트 작성 및 통과
- [ ] 인수 조건(AC) 전체 충족
- [ ] 회귀 테스트 통과

## 문서

- [ ] 필요한 경우 API 문서 / 사용자 가이드 갱신
- [ ] 변경 이력 기록

## 배포

- [ ] 스테이징 환경에서 정상 동작 확인
- [ ] 배포 파이프라인 통과

## 프로젝트별 추가 항목

(프로젝트 특성에 맞게 항목을 추가하거나 수정한다)

- [ ] (추가 항목 1)
- [ ] (추가 항목 2)
```

---

## 작성 규칙

1. 카테고리(코드 품질, 테스트, 문서, 배포)는 기본 제공하되, 프로젝트 특성에 맞게 조정 가능
2. 각 항목은 체크리스트(`- [ ]`) 형태로 검증 가능하게 작성
3. "프로젝트별 추가 항목" 섹션은 빈 체크리스트로 생성하여 사용자가 추후 커스터마이즈
4. 입력 문서에서 품질/테스트/배포 관련 요구사항이 언급되면 해당 항목을 DoD에 반영
