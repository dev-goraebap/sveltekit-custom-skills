# Sitemap 가이드

Sitemap은 PRD의 기능 요구사항(FR)과 UI 정보를 집약하여 **IA(Information Architecture) 원칙에 따라 설계한** 전체 화면 계층 구조 문서다.
단순한 화면 목록이 아니라, 사용자가 정보를 찾고 이동하는 구조 자체를 표현한다.

---

## 출력 경로

```
.blueprint/(<scope>)/design/sitemap.md
```

---

## 생성 시점

PRD 문서(`.blueprint/prd-*.md`)가 승인된 후, 사용자가 사이트맵 생성을 요청할 때.
PRD에 UI/화면 관련 기능 요구사항이 하나 이상 존재해야 생성한다.

---

## 도출 규칙

1. 모든 PRD 문서의 기능 요구사항(FR)에서 UI/화면 정보를 스캔한다
2. 화면을 사용자 역할(페르소나)별로 그룹화한다. PRD의 대상 사용자 정보를 기준으로 삼는다
3. MVP 포함 여부는 해당 FR의 우선순위가 Must이면 `Y`, 그 외는 `N`
4. 관련 FR은 해당 화면이 등장하는 FR ID를 모두 열거한다 (복수 가능)

---

## IA 설계 원칙 적용

Sitemap을 생성할 때 단순 나열이 아닌 **구조적 설계**를 수행한다:

### 계층 구조 설계
- 화면을 의미 있는 섹션으로 그룹화한다 (5±2개 원칙)
- 상위 섹션은 사용자 언어로 명명한다 (내부 용어 지양)
- 핵심 태스크까지 3단계(depth) 이내로 도달 가능하도록 구성한다

### 레이블링
- 화면명은 사용자가 이해하는 언어로 작성한다
- 같은 개념은 동일한 이름을 사용한다 (일관성)
- 모호한 이름("기타", "서비스")은 사용하지 않는다

### 페르소나 분리
- 서로 다른 사용자 그룹의 진입점과 주요 경로를 명확히 분리한다
- 권한이 다른 사용자의 경로가 불필요하게 겹치지 않도록 한다

---

## 업데이트 시점

- PRD가 수정되어 FR이 추가·변경·삭제될 때
- 사용자가 화면 구조 변경을 요청할 때
