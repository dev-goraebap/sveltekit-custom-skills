# Sitemap

> IA(Information Architecture) 원칙에 따라 설계한 화면 계층 구조.
> PRD 기능 요구사항에서 도출하며, 페르소나별 네비게이션 경로를 포함한다.

## 개요

| 항목 | 내용 |
|------|------|
| 총 화면 수 | N |
| MVP 포함 화면 | N |
| 최대 depth | N |

---

## 네비게이션 계층 구조

```
/ (홈)
├── [섹션명]  (/path)
│   ├── [화면명]  (/path/sub)
│   └── [화면명]  (/path/sub2)
├── [섹션명]  (/path2)
│   ├── [화면명]  (/path2/sub)
│   │   └── [화면명]  (/path2/sub/detail)
│   └── [화면명]  (/path2/sub2)
└── [섹션명]  (/path3)
    └── [화면명]  (/path3/sub)
```

> 같은 화면이 여러 경로에서 접근 가능한 경우 `→` 로 참조 표기:
> `└── [화면명]  → /path/to/screen`
