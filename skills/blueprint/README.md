### Lo-fi Design System. 그리고 사용자정의 소프트웨어 설계 지침 확장

구체적인 설계 가이드 문서는 다루지 않습니다만, 해당 스킬은 의도적으로 화면에 관련된 Lo-fi 디자인 설계 지침을 제공합니다. 이는 초기 MVP 단계가 마무리 될 때까지 사용자가 디자인, 사용자 경험 측면 보다 요구하는 기능을 검증하는데, 집중하기 위함입니다. 추후에 UI/UX를 개선해달라는 티케팅을 요청하여 새로운 테스크를 만들어 내고 개선하는 것이 가능합니다.

이미 ai-dev-task의 PRD와 task 문서로도 완성도 높은 소프트웨어를 만드는 것을 검증했지만, 일부 사용자는 요구사항정의서와 구현단계에서의 테스크 사이에 좀 더 명확한 가이드라인이 있길 바랄 수 있습니다. 이러한 문서는 대부분의 공통된 사례는 있을지 몰라도 여러 방법과 문서들이 존재하기 때문에 해당 스킬에서는 구체적으로 다루지 않습니다. 따라서 사용자가 PRD를 기준으로 별도의 프롬프트나 각 문서에 전문화된 스킬들을 통해서 설계 문서들을 만들어 추가로 문서를 구비하는 것이 가능합니다. 
이렇게 만들어진 문서는 구현단계의 지침에서 PRD 문서 외에 추가로 참조하여 할일, 하위 할일 들을 좀 더 구체적으로 만들어 냅니다.

- 각 테스크에 PRD의 FR ID를 추적할 수 있게 합니다.

단순히 사용자의 요청을 기록하는 것을 목표로 합니다. 구현하려던 테스크에 문제가 생겼거나, 어떤 개선요청등을 해서 AI 에이전트와 합의를 끝낸 시점에 에이전트가 먼저 해당 내용을 어떤 문서에 기록하여 의사결정 내용과 영향이 가는 FR, 테스크등의 변경이력을 관리합니다.
