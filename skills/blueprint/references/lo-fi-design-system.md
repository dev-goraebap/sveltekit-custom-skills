# Lo-fi Design System

MVP 단계에서 사용하는 최소한의 디자인 지침이다. 목표는 사용자가 기능을 검증할 수 있는 수준의 UI를 빠르게 만드는 것이며, 시각적 완성도는 의도적으로 후순위로 둔다.

## 원칙

1. **기능 우선** — 동작하는 기능이 예쁜 UI보다 먼저다
2. **일관된 토큰** — 색상·간격·폰트 크기를 토큰으로 정의하고 임의 값을 쓰지 않는다
3. **시스템 기본값 활용** — 시스템 폰트, 프레임워크 기본 컴포넌트를 우선 사용한다
4. **장식 금지** — 그림자, 그라데이션, 애니메이션, 커스텀 아이콘은 MVP에서 사용하지 않는다
5. **접근성 최소 보장** — Lo-fi라도 읽을 수 없거나 조작할 수 없으면 검증 자체가 불가능하다

---

## 1. 색상

흑백 기반 + 강조색 1개. 모든 색상은 아래 토큰만 사용한다.

| 토큰 | 값 | 용도 |
|------|----|------|
| `bg` | `#FFFFFF` | 페이지 배경 |
| `surface` | `#F9FAFB` | 카드, 섹션 배경 |
| `border` | `#E5E7EB` | 구분선, 입력 테두리 |
| `text-primary` | `#111827` | 본문, 제목 |
| `text-secondary` | `#6B7280` | 부가 설명, 플레이스홀더 |
| `accent` | `#2563EB` | 링크, 주요 버튼, 활성 상태 |
| `accent-hover` | `#1D4ED8` | 주요 버튼 hover/press |
| `danger` | `#DC2626` | 에러 메시지, 삭제 버튼 |
| `success` | `#16A34A` | 성공 메시지 |

**규칙:**
- 위 토큰 외 색상을 사용하지 않는다
- 프로젝트 브랜드 컬러가 있으면 `accent`만 교체한다
- 다크모드는 MVP에서 구현하지 않는다

---

## 2. 타이포그래피

시스템 폰트 스택을 사용한다. 커스텀 웹폰트는 로딩하지 않는다.

```
font-family: system-ui, -apple-system, sans-serif
```

| 역할 | 크기 | 굵기 | 용도 |
|------|------|------|------|
| `heading-lg` | 28px | 700 (Bold) | 페이지 제목 |
| `heading-md` | 22px | 600 (Semi) | 섹션 제목 |
| `heading-sm` | 18px | 600 (Semi) | 카드 제목, 서브 섹션 |
| `body` | 16px | 400 (Regular) | 본문 텍스트 |
| `body-sm` | 14px | 400 (Regular) | 부가 설명, 테이블 셀 |
| `caption` | 12px | 400 (Regular) | 타임스탬프, 힌트 (본문에는 사용 금지) |

**규칙:**
- 줄 높이(line-height)는 본문 1.5, 제목 1.3
- 줄 길이는 최대 72자 (가독성 한계)
- 위 6단계 외 크기를 사용하지 않는다

---

## 3. 간격

4px 기반 스케일. 모든 패딩·마진·갭에 이 값만 사용한다.

| 토큰 | 값 | 주 용도 |
|------|----|---------|
| `space-1` | 4px | 아이콘-텍스트 간격, 인라인 요소 |
| `space-2` | 8px | 관련 요소 사이 (라벨-입력) |
| `space-3` | 12px | 리스트 항목 사이 |
| `space-4` | 16px | 컴포넌트 내부 패딩 |
| `space-6` | 24px | 섹션 내부 여백 |
| `space-8` | 32px | 섹션 간 구분 |
| `space-12` | 48px | 페이지 상단/하단 여백 |

---

## 4. 레이아웃

| 항목 | 규격 |
|------|------|
| 최대 콘텐츠 너비 | 960px (중앙 정렬) |
| 페이지 좌우 패딩 | 16px (모바일), 24px (태블릿 이상) |
| 기본 구조 | 단일 컬럼 |
| 그리드 (필요 시) | 최대 2~3 컬럼, 갭 16px |
| 반응형 브레이크포인트 | 768px 하나만 사용 (모바일 / 데스크톱) |

**규칙:**
- 복잡한 멀티 컬럼 레이아웃은 피한다
- 모바일에서는 항상 단일 컬럼으로 쌓는다
- 가로 스크롤이 발생하지 않아야 한다

---

## 5. 컴포넌트

프레임워크 기본 컴포넌트를 사용하되, 아래 최소 스펙을 유지한다. 값은 위 토큰을 사용한다.

### 버튼

| 속성 | Primary | Secondary | Danger |
|------|---------|-----------|--------|
| 배경 | `accent` | `transparent` | `danger` |
| 텍스트 | `#FFFFFF` | `accent` | `#FFFFFF` |
| 테두리 | 없음 | 1px `border` | 없음 |
| 패딩 | 8px 16px | 8px 16px | 8px 16px |
| 라운딩 | 6px | 6px | 6px |
| 최소 높이 | 40px | 40px | 40px |
| 비활성 | opacity 0.5, 클릭 불가 | 동일 | 동일 |

### 입력 (Input, Textarea, Select)

| 속성 | 값 |
|------|----|
| 테두리 | 1px `border` |
| 포커스 테두리 | 2px `accent` |
| 패딩 | 8px 12px |
| 라운딩 | 6px |
| 최소 높이 | 40px |
| 라벨 | 입력 위에 `body-sm` 굵기 500으로 표시 (placeholder만으로 대체 금지) |
| 에러 | 테두리 `danger`, 아래에 `danger` 색상 메시지 |

### 카드

| 속성 | 값 |
|------|----|
| 배경 | `surface` |
| 테두리 | 1px `border` |
| 패딩 | 16px |
| 라운딩 | 8px |
| 그림자 | 없음 |

### 테이블

| 속성 | 값 |
|------|----|
| 헤더 배경 | `surface` |
| 헤더 텍스트 | `text-primary`, `body-sm` 굵기 600 |
| 셀 패딩 | 8px 12px |
| 행 구분 | 하단 1px `border` |
| 호버 | 없음 (MVP에서 생략) |

### 배지 / 태그

| 속성 | 값 |
|------|----|
| 패딩 | 2px 8px |
| 라운딩 | 4px |
| 폰트 | `caption`, 굵기 500 |
| 스타일 | 배경 `surface`, 텍스트 `text-primary`, 테두리 1px `border` |
| 상태 강조 시 | 배경에 해당 토큰 색상 10% 적용 (예: accent 배경에 accent 텍스트) |

## 6. 상태 패턴

기능 검증에 필수적인 UI 상태는 반드시 구현한다.

| 상태 | 표현 방식 |
|------|-----------|
| **로딩** | 텍스트 "로딩 중..." 또는 간단한 스피너. 스켈레톤 불필요 |
| **빈 상태** | 중앙 정렬 텍스트 + 안내 메시지 (예: "아직 항목이 없습니다") + 액션 버튼 (있으면) |
| **에러** | 해당 영역 근처에 `danger` 색상 텍스트로 표시. 원인 + 해결 방법 포함 |
| **성공** | 짧은 텍스트 메시지 (토스트 또는 인라인). 3~5초 후 자동 사라짐 |
| **비활성** | opacity 0.5 + 커서/인터랙션 비활성 |
| **확인 대화상자** | 삭제 등 파괴적 동작 전에 반드시 확인 요청 |

---

## 7. 접근성 최소 기준

Lo-fi 단계에서도 아래 항목은 반드시 지킨다.

- **명암비**: 텍스트와 배경 간 4.5:1 이상 (위 토큰 조합은 이미 충족)
- **라벨**: 모든 입력에 보이는 라벨 필수 (placeholder만으로 대체 금지)
- **키보드**: Tab으로 모든 인터랙티브 요소 접근 가능, 포커스 링 제거 금지
- **클릭 영역**: 버튼·링크의 최소 클릭/터치 영역 40×40px
- **색상 단독 금지**: 에러·성공 상태를 색상만으로 구분하지 않음 (텍스트 병행)

---

## 하지 않는 것

- 커스텀 아이콘 세트, 일러스트레이션 제작
- 다크모드, 테마 시스템
- 마이크로 인터랙션, 트랜지션 애니메이션
- 반응형 브레이크포인트 세분화 (768px 하나만 사용)
- 그림자, 그라데이션, 블러 효과

## 개선 시점

MVP 기능 검증이 완료된 후, 이슈 트래킹을 통해 UI/UX 개선 태스크를 등록한다. 이 시점에 Lo-fi 지침을 해제하고 본격적인 디자인 시스템을 적용한다.
