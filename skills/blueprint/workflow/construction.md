# 작업 항목 생성 (Construction)

## 목표

PRD와 설계 문서를 기반으로 구현 가능한 작업 항목(Work Item) 목록을 생성한다. MVP 범위를 확정하고, 각 작업 항목이 PRD의 기능 요구사항(FR)을 추적할 수 있도록 한다.

## 입력

- `blueprint/prd-[프로젝트명].md` (필수)
- `blueprint/design/` 폴더의 모든 문서 (선택 — 있으면 참조)

## 출력

- **형식:** Markdown (`.md`)
- **위치:** `blueprint/`
- **파일명:** `work-items.md`

## 프로세스

1. **문서 읽기:** `blueprint/prd-*.md` 파일과 `blueprint/design/` 폴더의 문서를 모두 읽는다.
2. **상위 작업 항목 생성:** 기능 요구사항(FR)을 분석하여 주요 상위 작업 항목을 생성한다. 대략 5~8개가 적절하다.
3. **MVP 범위 확정:** 사용자에게 MVP에 포함할 상위 작업 항목을 확인받는다. MVP 항목을 목록 상단에 배치한다.
4. **확인 대기:** 사용자가 MVP 범위를 확인할 때까지 하위 작업 항목을 생성하지 않는다.
5. **하위 작업 항목 생성:** 확인 후 각 상위 작업 항목을 실행 가능한 하위 작업 항목으로 분해한다.
6. **관련 파일 식별:** 생성하거나 수정해야 할 파일을 식별하여 나열한다.
7. **저장:** `blueprint/work-items.md`에 저장한다.

## 출력 형식

```markdown
# Work Items

## 관련 파일

- `path/to/file.ts` - 간략한 설명
- `path/to/file.test.ts` - 테스트 파일

## 완료 안내

각 하위 작업 항목을 완료할 때마다 `- [ ]`를 `- [x]`로 변경하여 진행 상황을 추적한다.

## 작업 항목

### MVP

- [ ] WI-01 상위 작업 항목 제목 → `FR-01, FR-02`
  - [ ] WI-01.1 하위 작업 항목 설명
  - [ ] WI-01.2 하위 작업 항목 설명
- [ ] WI-02 상위 작업 항목 제목 → `FR-03`
  - [ ] WI-02.1 하위 작업 항목 설명

### 후속

- [ ] WI-03 상위 작업 항목 제목 → `FR-05, FR-06`
  - [ ] WI-03.1 하위 작업 항목 설명
```

## 핵심 규칙

- 각 작업 항목에 `WI-01`, `WI-02`, ... 형식으로 고유 ID를 부여한다. 하위 항목은 `WI-01.1`, `WI-01.2` 형식이다.
- 상위 작업 항목 제목 끝에 `→`로 관련 FR ID를 연결한다 (예: `→ FR-01, FR-02`)
- MVP 항목을 `### MVP` 섹션에, 나머지를 `### 후속` 섹션에 배치한다
- WI 번호는 MVP/후속 구분 없이 전체에서 순차적으로 매긴다
- 설계 문서가 있으면 하위 작업 항목을 더 구체적으로 작성한다
- 작업 항목은 점진적으로 추가될 수 있다 (티케팅을 통해 새 항목이 생길 수 있음)

## 대상 독자

작업 항목의 주요 독자는 기능을 구현할 **주니어 개발자**라고 가정한다. 각 하위 항목은 무엇을 해야 하는지 명확히 알 수 있어야 한다.
