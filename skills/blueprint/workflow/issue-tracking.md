# 이슈 추적 (Issue Tracking)

## 목표

구현 중 발생하는 문제, 개선 요청, 의사결정 내용을 `blueprint/issues.md`에 기록한다. AI 에이전트가 사용자와 합의한 시점에 자동으로 기록하여, 변경이력과 영향받는 FR·작업 항목을 추적한다.

## 출력

- **형식:** Markdown (`.md`)
- **위치:** `blueprint/`
- **파일명:** `issues.md`

## 기록 시점

다음 상황에서 에이전트가 **먼저** 이슈 기록을 제안한다:

1. **구현 중 문제 발생:** 작업 항목 진행 중 기술적 문제나 요구사항 충돌이 발견된 경우
2. **개선 요청:** 사용자가 기존 기능의 UI/UX 개선, 리팩토링 등을 요청한 경우
3. **요구사항 변경:** PRD의 FR이 변경되거나 새로운 FR이 추가된 경우
4. **의사결정:** 구현 방향에 대해 사용자와 합의가 이루어진 경우

## 프로세스

1. 사용자와 합의가 완료된 시점에 에이전트가 이슈 기록을 제안한다
2. 사용자가 승인하면 `blueprint/issues.md`에 이슈를 추가한다
3. 영향받는 FR이나 작업 항목이 있으면 해당 문서도 함께 업데이트한다

## 출력 형식

```markdown
# Issues

## ISS-001: [이슈 제목]

- **유형:** 버그 | 개선 | 변경 | 결정
- **날짜:** YYYY-MM-DD
- **상태:** 열림 | 완료
- **영향:** `FR-03`, `WI-02.1`

### 내용

합의된 내용을 간결하게 기록한다.

### 조치

- 변경한 사항 또는 앞으로 해야 할 사항을 나열한다.

---
```

## 핵심 규칙

- 이슈 ID는 `ISS-001`부터 순차적으로 부여한다
- 영향받는 FR ID와 작업 항목(WI) 번호를 반드시 명시한다
- 기록은 간결하게 — 합의 내용과 조치사항 중심으로 작성한다
- 에이전트가 먼저 기록을 제안하되, 사용자 승인 후 기록한다
